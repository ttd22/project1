Continuous Integration and Deployment

What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is simply automate the testing and the deployment. In the past the development and the deployment cycle took a lot of effort and time of developers to develop the software and deploy it into production or the live environment. The process in the past not only takes lots of time but also comes up with lots of human intervention and as a result lots of human errors arrive.

Deploying with GitHub Actions

Development workflows

  • Trigger my deployment. It runs whenever there are pull requests

  • Set up jobs for development workflows. A job is a set of steps in a workflow that execute on the same runner.

    Before merging pull requests, the code is reviewed with the jobs run_pytest and deploy_heroku. Both jobs run on the runner ubuntu-latest. A runner is a server that runs your workflows when they're triggered. The job deploy_heroku will deploy to Heroku dev app

    In your newly created Github repository, add new repository secrets for DOCKER_USERNAME, DOCKER_PASSWORD, HEROKU_API_KEY (Values are DOCKER_USERNAME: your docker hub username; DOCKER_PASSWORD: your docker hub password; HEROKU_API_KEY: API key from the heroku app)

    GitHub Notes: Set the action secrets repository in: -> settings -> actions -> secrets

    Create an account with Heroku, create an app for production and an app for development and add the information of heroku app name and heroku email

    Heroku Notes: Get the heroku API key from account in: -> applications -> create authorization button

    Create a repository on Docker Hub. Add your images under "Extract metadata (tags, labels) for Docker"

Production workflows

  • Trigger my deployment. It runs whenever there are pushes to project, push to branch master

  • Set up jobs for production workflows. A job is a set of steps in a workflow that execute on the same runner.

    In the production workflow, there are jobs run_pytest, build-container and deploy. All jobs run on the runner ubuntu-latest. A runner is a server that runs your workflows when they're triggered.

    In your newly created Github repository, add new repository secrets for DOCKER_USERNAME, DOCKER_PASSWORD, HEROKU_API_KEY (Values are DOCKER_USERNAME: your docker hub username; DOCKER_PASSWORD: your docker hub password; HEROKU_API_KEY: API key from the heroku app)
